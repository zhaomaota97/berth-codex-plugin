# Migration fidelity guide

Use the source Agent as the behavior baseline. Inventory all material capabilities before editing, then map each capability to the generated Package.

Hard assertions always outrank aggregate similarity scores. Missing a critical Tool, approval boundary, attachment parser, required schema, or artefact blocks publication.

Do not demand byte-identical model output. Compare required facts, structure, actions, approvals, files, and business outcome. Semantic evaluation may supplement these checks but cannot override a hard failure.

If the source Agent cannot be executed, report dynamic fidelity as unverified and require human confirmation.
